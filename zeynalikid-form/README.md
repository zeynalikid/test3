# زینالیکید — پروژه ثانویه (B) — فرم مشاوره

این پروژه فقط شامل **فرم مشاوره** و **صفحه موفقیت** است. پنل مدیریت در پروژه اصلی (A) قرار دارد.

## صفحات
| view | صفحه | توضیح |
|---|---|---|
| `form` | FormPage | فرم مشاوره (موضوع، والد، فرزند، تغذیه و سلامت) |
| `success` | SuccessPage | پیام موفقیت + انتخاب بازه تماس + دکمه دوره‌ها |

## راه‌اندازی
```bash
npm install
cp .env.example .env   # سپس مقادیر را وارد کنید
npm run dev
npm run build
```

## متغیرهای محیطی
| متغیر | توضیح | پیش‌فرض |
|---|---|---|
| `VITE_SUPABASE_URL` | آدرس Supabase (مشترک با پروژه اصلی) | — |
| `VITE_SUPABASE_ANON_KEY` | کلید anon | — |
| `VITE_APP_A_URL` | آدرس پروژه اصلی (A) — دوره‌ها + پنل مدیریت؛ دکمه «معرفی و ثبت مستقیم دوره» | `https://zeynalikid.vercel.app` |

> توجه (اصلاح ۱۸): نام‌گذاری پروژه‌ها اصلاح شد — این پوشه اکنون **پروژه ثانویه (B)** نام دارد و متغیر `VITE_APP_A_URL` به آدرس **پروژه اصلی (A)** یعنی دوره‌ها + پنل مدیریت اشاره می‌کند.

## هماهنگی با پروژه اصلی (A)
- تنظیمات (تم، ترجمه‌ها، فیلدها، ...) هنگام بارگذاری از جدول `settings` سوپابیس خوانده می‌شود؛ تغییرات پنل مدیریت (در پروژه اصلی) اینجا اعمال می‌شود.
- ثبت فرم مشاوره با `createSubmission` در جدول `submissions` ذخیره می‌شود و در پنل مدیریت پروژه اصلی قابل مشاهده است.
- زبان با کلید `zkid_lang` در localStorage بین دو پروژه هماهنگ است.
- اگر Supabase تنظیم نشده باشد، همه‌چیز با localStorage کار می‌کند.
